What's new in version 2.01.02

DataWedge provides a convenient mechanism for non-scanning aware applications to receive data from embedded or connected data capture devices. DataWedge v2.01.02 builds on v2.00.00 to include the following new features...

  • New Test Display Window with interchangeable Text/Hex formats and extra information
  • New Log Display Window with scrolling
  • Improved DataWedge Startup Time when MSR and RFID are present
  • Gen2 RFID Support is added.
  • Several other bug fixes:
    • Device Enable/Disable persistance over cold boot
    • Virtual Key Support and few other issues in Prefix/Suffix
    • Smaller Controls for Feedback Parameters in order to accommodate broader edit boxes.
    • Enhancements in MSR reading
    • Enable Serial Interface on startup
    • Event Based Serial Reading Capability added removing earlier polling based mechanism.
    • Delays regarding RFID are fixed
    • Double quote character in barcodes was scanned as Single quote
    • Tilda (~) character in barcodes caused abnormal behavior over Remote Desktop Client
    • Fixed Data Corruption issue in Binary mode ("Don't treat Data as Text"), where when you exit out of DW in Binary mode and run it later scanned barcodes got corrupted.
    • CAPSLOCK state caused barcode alpha characters to change case (See Known Issues for some inconsistency in Windows Mobile)
    • Group Seperator character (0x1D) could not be sent to Remote Desktop.

For the latest software updates, go to the Symbol Developer Zone web site at:

http://devzone.symbol.com

Installation

To install DataWedge so that it is launched automatically after a cold boot (clean boot for WM2005) , do the following:

  1. Unzip the DataWedge download package into a temporary folder on your PC
  2. If installing on a Pocket PC device, copy "\Bin\PPC\DataWedge.exe" from your PC to the \Application folder on the device
  3. If installing on a Windows CE device, copy "\Bin\WCE\DataWedge.exe" from your PC to the \Application folder on the device
  4. Copy "\Application\DataWedge.cpy" from your PC to the \Application folder on the device
  5. Copy "\Application\Startup\DataWedge.run" from your PC to the \Application\Startup folder on the device
  6. Cold boot (clean boot for WM2005) the device

Usage Notes

  1. MSR API - In order to support the MSR, DataWedge requires the new MSR API (MSRAPI32.DLL)
  2. MC 1000 - The user does not have access to the UI when using DataWedge on the MC 1000
  3. RFID - Support for RFID should be considered as beta; see known issues below
  4. MK 2000 - Support for the MK 2000 in this version is in beta although there are no known issues

Device Updates

Before using DataWedge with your mobile computer you may need to update certain drivers. Refer to the DriverUpdateInfo.txt file located in "\Program Files\Symbol Mobility Developer Kit for eVC4\v1.4\DriverUpdates" for more information about the individual files that can be updated and what devices are supported.

MSRAPI32 Support

In the event that a device in the field has not been updated to the latest MSR software, you can update your device by copying the following files to the \Platform directory.

  • MsrApi32.dll
  • MagStripe.cpy
  • MagStripe.reg

NOTE: A cold boot (clean boot for WM2005) must be performed after any update to ensure the files have been installed properly.

NOTE: In Windows Mobile 2005, the \Platform does not exist. Hence the above files should be copied to \Application folder and .cpy file should be edited appropriately.

Known issues

  1. RFID support can be affected by hardware and firmware versions
  2. On RFID devices with Platform version Plat0301074 DataWedge may not exit correctly; requiring a warm boot
  3. On RFID devices with internal WJ module DataWedge may intermittently fail to detect/open the RFID module
  4. The beep time in the feedback settings may not change the length of the beep on PDT8100 WM2003 devices
  5. The barcode feedback menu may be missing the beep time and frequency fields on MC50 WM2003 devices
  6. MSR Feedback Settings can’t be changed effectively. Watever the value we keep, the same parameter the hardware would work at.
  7. Barcode Feedback Setting "LED Time" can’t be changed effectively.
  8. The '\f' and '\n' prefix/suffix characters may be ignored on CE4.2 devices
  9. In Auto-PIE mode, after PIE has popped up and KeyBoard is on, if you kill PIE KeyBoard area becomes inactive. User should Hide the KeyBoard via "Hide Input Panel".
  10. MC 3000 Dialog windows may appear imbalanced and too much oriented to the left side of the screen.
  11. In WM when you connect to the TSC and then Enable CAPSLOCK, then type few characters while CAPS lock is on, then Close TSC with the X on right hand corner and Then again invoke TSC and scan a barcode, this will result in case Inversion.
  12. If you check the option "Send keys only to Pocket IE" at least for one scanner type, data from all four scanner types are diverted to Pocket IE.
  13. If you select Serial Option "Send data after X seconds" and set its timer to more than 2 seconds the data would not be sent.
